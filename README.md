# NETPRACTICE 🔌

## DESCRIPTION

NetPractice 🔌 is a 42 networking project focused on understanding how small networks are configured using **TCP/IP** addressing. 
The goal is to solve **10 levels** of networking exercises by correctly setting *IP addresses*, *subnet masks*, *default gateways*, and *routes* so that all devices can communicate properly. 
The project helps build a **practical understanding** of how routers, switches, and network segmentation work together.

## INSTRUCTIONS

### Prerequisites
- Access to the 42 NetPractice training interface.
- A browser to complete the levels online.

### How to run the training interface
1. Open the NetPractice training interface from your 42 intranet.
2. Log in with your 42 credentials.
3. Solve each level by adjusting the network configuration.

### How to export configurations
- After solving a level, click **Get my config** to export that level’s configuration file.
- Repeat this for all 10 levels.
- Save each exported file with the corresponding level number.

### Submission requirements
- You must place **10 exported configuration files** in the **root of the repository**, one file per level.
- Make sure all levels are completed and all exported configs are included before submission.

## RESOURCES

### Networking concepts studied
This project covers core networking concepts such as *TCP/IP* addressing, *IPv4*, subnet *masks*, *CIDR* notation, *private IP ranges*, *loopback addresses*, *routers*, *switches*, and the *OSI/TCP-IP* model.
